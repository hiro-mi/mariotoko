# 実装計画

このディレクトリでは、これから行う実装計画と、実装済みの計画記録を管理します。恒久的なゲーム仕様は `docs/` 直下の各仕様書に置き、実装単位ごとの作業計画はここに置きます。

## ディレクトリ

- `active/`: これから実装する計画、または実装中の計画。
- `completed/`: 実装が完了した計画と、その結果記録。

## 作成ルール

- 実装に入る前に、`active/NNNN-topic.md` の形式で計画を書く。
- `NNNN` は4桁の連番にする。
- 計画には、目的、実装内容、対象ファイル、確認項目を含める。
- 実装中に計画から外れる重要判断をした場合は、関連する仕様書または計画ドキュメントに反映する。

## 完了時のルール

実装が完了したら、計画ファイルを `completed/` へ移動します。移動後のファイルには、次を追記します。

- 完了日
- 実装結果
- 確認したこと
- 未完了のことや次回課題

実装で恒久仕様が固まった場合は、計画ファイルだけでなく、`game-design.md`、`gameplay-rules.md`、`content-spec.md`、`technical-design.md`、`verification.md` の該当箇所にも反映します。
