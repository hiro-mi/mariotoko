# AGENTS.md

## プロジェクト憲章

このプロジェクトは、ブラウザで動作する2D横スクロールアクションゲーム「スーパーマリ漢＜OTOKO＞」を制作するためのものです。

「スーパーマリ漢＜OTOKO＞」は、クラシックな横スクロールジャンプアクションの文法を参考にしたオリジナル作品として実装してください。既存作品の体験に敬意を払いながらも、画像、音楽、効果音、ロゴ、キャラクター、ステージ配置、固有設定は流用しないでください。

最大の独自性は、ファイヤーボールの代わりに「漢ボール」が飛ぶことです。漢ボールは画面内に最大2つまで存在できます。

## 基本方針

- プロジェクト名およびゲーム題名は「スーパーマリ漢＜OTOKO＞」とします。
- ゲーム本体は、オンライン接続なしで動作するローカルHTMLゲームとして作ります。
- 初期実装は、ビルド不要のプレーンなHTML/CSS/JavaScriptを基本とします。
- `index.html` をブラウザで直接開いて遊べる状態を優先します。
- CDN、外部API、オンライン画像、オンライン音声、Webフォントなど、ネットワーク依存のある構成は使わないでください。
- 画像、音声、フォントなどのリソースが必要な場合は、すべてリポジトリ内のローカルファイルとして配置してください。

## ゲーム仕様の基本

- おおむねクラシックな横スクロールジャンプアクションとして設計します。
- プレイヤーは走る、跳ぶ、敵を踏む、パワーアップする、といった基本操作を持ちます。
- 遠距離攻撃として、ファイヤーボールではなく「漢ボール」を撃ちます。
- 漢ボールは画面内に最大2つまで存在できます。3つ目を撃とうとした場合の挙動は、ゲーム仕様側で明確に定義してください。
- スタート画面には、マリ漢がたどってきた足跡を3行で表示してください。
- ステージ、敵、アイテム、演出、UIは、既存作品の直接的な複製ではなく、この作品独自の表現として作ってください。

## 日本語運用

- コードコメントは基本的に日本語で書いてください。
- UI文言、ドキュメント、プルリクエスト本文、レビュー返信は基本的に日本語で書いてください。
- コミットメッセージも、特別な理由がない限り日本語を優先してください。
- 外部仕様名、API名、ライブラリ名、ファイル名、識別子など、日本語化すると不自然または不正確になるものは英語のまま扱ってください。

## 権利と表現の注意

- 既存作品の画像、音楽、効果音、ロゴ、キャラクター、ステージ、固有設定を流用しないでください。
- 既存作品の名称や商標に強く依存したUI、素材、文章は避けてください。
- 「スーパーマリ漢＜OTOKO＞」は仮タイトルとして扱い、ビジュアル、音、物語、キャラクター設定はオリジナルにしてください。
- クローン的な遊び心は保ちつつ、素材と設定はオリジナル作品として成立するものにしてください。

## 作業時の更新ルール

- 実装に入る前に、作業計画を `docs/plans/active/` に追加してください。
- 実装が完了したら、該当する計画を `docs/plans/completed/` に移動し、実装結果と確認内容を追記してください。
- ゲーム仕様を変更した場合は、関連する `docs/` のドキュメントも更新してください。
- 実装方式やディレクトリ構成を変更した場合は、`docs/technical-design.md` を更新してください。
- 動作確認項目や完成条件を変更した場合は、`docs/verification.md` を更新してください。
- 作業者全体に関わる恒久的な方針を変更する場合のみ、この `AGENTS.md` を更新してください。
